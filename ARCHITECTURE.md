Контекст: Я владелец автоматизированной SaaS-системы по генерации и публикации YouTube Shorts. Моя текущая успешная ниша — "JobHack AI" (поиск работы в IT). Я хочу масштабировать эту систему на новые ниши. Ниже описана полная архитектура моего конвейера и воронки продаж. Изучи её, чтобы давать мне точные инструкции по масштабированию, написанию кода или созданию контента.

🎯 1. Бизнес-модель и Воронка (Funnel)
Трафик: Органический трафик из YouTube Shorts.

Перелив: В каждом видео есть призыв перейти по ссылке в профиле канала.

Конверсия: В профиле YouTube закреплены 2 ссылки: главная — на Telegram-бота (выдает лид-магнит или функционал, например @JobHackAIBot), вторая — на Telegram-канал.

Цель: Сбор базы в TG-боте и канале для дальнейшей монетизации.

🏭 2. Архитектура системы (Конвейер 24/7)
Система работает на Linux-сервере (VPS) круглосуточно через виртуальные экраны утилиты screen. Она состоит из двух независимых скриптов (Повар и Курьер), связанных через Google Таблицу.

Модуль A: База данных (Google Таблицы)

Роль: Панель управления контентом.

Структура: Содержит колонки для заголовка, описания, текста для озвучки и статуса.

Статусы жизненного цикла: NEW (ждет создания) -> DONE (видео срендерено) -> SCHEDULED (загружено на YouTube).

Надежность: Встроена система retry (повторных попыток) на случай ошибок 500 от серверов Google.

Модуль B: Генератор Видео (shorts_maker.py)

Роль: Сборка визуала и звука. Работает в цикле while true с паузой в 10 минут. Берет строки со статусом NEW.

Озвучка: Использует API OpenAI (модель tts-1-hd, голоса nova или fable) либо локальный Edge-TTS (бесплатно, быстро, например ru-RU-DmitryNeural или ru-RU-SvetlanaNeural).

Тайминг субтитров: Пропорциональная математика по буквам для идеального синхрона голоса и текста. Узнает длину аудио, считает буквы, раздает время каждому слову.

Визуал (MoviePy + ImageMagick): Берет случайное фоновое видео из папки backgrounds/. Накладывает случайную музыку из папки music/.

Дизайн текста: Шрифт системный Arial-Black, цвет желтый (#FFEA00), позиция 75% высоты экрана (внизу по центру).

Предохранитель: Алгоритм авто-масштабирования проверяет ширину фразы и сжимает длинные слова до 90% ширины экрана, чтобы они не вылезали за края.

Финал: Сохраняет .mp4 в папку ready_videos/ и меняет статус в таблице на DONE.

Модуль C: Авто-постер (autoposter.py)

Роль: Выгрузка готовых видео на канал. Работает в фоне, проверяя таблицу каждые 60 секунд.

Логика: Ищет строки со статусом DONE и наступившим временем публикации.

YouTube Data API v3: Автоматически подтягивает заголовок, описание и хештеги из таблицы. Выставляет категорию, пометку "не для детей" и статус отложенной публикации. Авторизация через OAuth 2.0 с автоматическим обновлением токенов.

Финал: Меняет статус в таблице на SCHEDULED.

Модуль D: Безопасность

Все ключи (credentials.json, token.json, client_secrets.json, API ключи) спрятаны в .gitignore.

🚀 3. Инструкция: Как запустить новую нишу за 30 минут
Для запуска нового проекта (например, Crypto, Психология, Факты) я выполняю следующие шаги:

Клонирование: Копирую папку проекта (кроме файлов в ready_videos, аудио и .git) в новую директорию (например, crypto_shorts).

Замена медиа: Удаляю старые видео из backgrounds/ и музыку из music/. Загружаю 5-10 новых тематических фонов и треков.

База данных: Создаю новую Google Таблицу, даю доступ сервисному аккаунту (или копирую ID для OAuth) и вставляю ID в код.

Воронка: Создаю нового Telegram-бота и TG-канал под нишу.

Сценарии: Генерирую 30-дневный контент-план (Крючок, Мясо, Призыв в бота) и вставляю в Таблицу.

Смена голоса: В shorts_maker.py меняю голос под нишу (например, на ru-RU-SvetlanaNeural или другой в OpenAI).

Авторизация YouTube: Удаляю старый token.json в новой папке. При первом запуске скрипт запросит логин в новый YouTube-канал. Оформляю канал (Аватар, Баннер, Ссылки на нового бота).

Запуск: Открываю 2 новых окна screen на сервере и запускаю shorts_maker.py и autoposter.py в бесконечном цикле.
